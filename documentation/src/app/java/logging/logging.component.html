Logging (Java)

>> Please don't use System.out.println() <<

When using any of the abstractions provided, they all extend from a single base class called Loggable.

We have created simple helper methods, that provide consistent console output for all levels of logging, however you also have access to log Which allows you to control your own logging if you don't want to use the pre-defined logging methods available.

There are a number of logging methods, for each standardized level

  1. logInfoMessage(String emoji, String message, String value) - INFO level message
  2. logErrorMessage(String message, String value) - ERROR level message.
  3. logDebugMessage(String message, String value) - DEBUG level message.
  4. logDebugMessage(String message) - DEBUG level message.
  5. logTraceMessage(String message, String value) - TRACE level message.
  6. logWarnMessage(String message) - WARN level message.

Using standard logging

If you don't want to use the methods supplied, for what ever reason - you can access this.log when using AbstractBase as your superclass. When using this.log, you actually have a reference to org.slf4j.Logger.
